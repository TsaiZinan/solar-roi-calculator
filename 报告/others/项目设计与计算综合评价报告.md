**评审日期**: 2026-04-29

## 一、项目概述

本项目用于计算广东汕头市雅威机电实业 0.12MW / 0.257MWh 工商储项目的光储充一体化系统每日收益、历史汇总收益和全年收益预测。

核心输入为阳光云导出的 5 分钟间隔 CSV 日报表（含光伏发电功率、储能有功功率、负载功率、SOC 等字段），辅以光伏曲线截图的 OCR 提取、电网电价表和充电桩定价表。输出为逐日 Markdown 报告、总收益汇总表、ROI 分析报告和储能增量拆解报告。

### 2.2 设计亮点

1. **配置中心化**: `config.py` 作为唯一配置入口，避免了参数散落在各脚本中的问题。
2. **电价版本化**: `电网电价.csv` 和 `充电桩定价.csv` 支持按"记录日期"自动选择最新电价，天然支持电价政策变更历史追踪。
3. **口径一致性保障**: `regenerate_reports.py` + `init_summary.py` 形成了"口径变更 → 全量重算 → 总表重建"的标准流程，确保日报、总表和 ROI 报告在任何时刻口径一致。
4. **幂等设计**: 每次运行 `calc_revenue.py` 都会完整重写日报文件并重建总表，不存在增量追加导致的累积误差。
5. **版本演进有据**: README 中详细记录了从 1.0 到 1.7 的完整演进历史，包括两次关键 Bug 修复的根因、影响范围和修复动作。

## 三、核心计算逻辑评价

### 3.1 真实总负载定义

**代码位置**: [calc_revenue.py L63-66](file:///Users/cai/SynologyDrive/Project/#ProjectWork-000000-光伏收益计算/Script/calc_revenue.py#L63-L66)

**公式**: `真实总负载 = 光伏发电功率 - CSV中的负载功率`

**背景**: 阳光云导出的 CSV 中，"负载功率"字段并非真正的用电负载，而是一个净功率值（正数表示光伏余电，负数表示缺电需从电网/储能补充）。因此用 `光伏发电 - 负载功率` 还原真实用电量是正确的。

**评价**: 逻辑自洽，与实际数据验证结果一致。但 CSV 列名 `负载功率` 容易造成误解，建议在代码注释中明确说明这个等式与直觉的偏差。

### 3.2 能量流向拆分模型

**代码位置**: [calc_revenue.py L80-101](file:///Users/cai/SynologyDrive/Project/#ProjectWork-000000-光伏收益计算/Script/calc_revenue.py#L80-L101)

模型将系统中所有能量流向拆分为 **12 条路径**:

| 来源 | 去向 | 变量 |
|---|---|---|
| 光伏 → 工厂 | `pv_to_factory` | 直供工厂替代购电 |
| 光伏 → 充电桩 | `pv_to_ev` | 直供充电桩形成收入 |
| 光伏 → 储能 | `pv_to_ess` | 余电充入储能 |
| 光伏 → 电网 | `pv_to_grid` | 余电上网售出 |
| 储能 → 工厂 | `ess_to_factory` | 放电供工厂 |
| 储能 → 充电桩 | `ess_to_ev` | 放电供充电桩 |
| 储能 → 电网 | `ess_to_grid` | 放电余量上网 |
| 电网 → 工厂 | `grid_to_factory` | 购电供工厂 |
| 电网 → 充电桩 | `grid_to_ev` | 购电供充电桩 |
| 电网 → 储能 | `grid_to_ess` | 购电充储能 |

**优先级规则**:
1. 光伏优先满足工厂 → 再充电桩 → 剩余充储能 → 最后上网
2. 储能放电优先供工厂 → 再充电桩 → 剩余上网
3. 电网补足所有剩余缺口

**评价**: 流向覆盖完整，优先级设置合理（光伏自发自用优先于储能充放电，储能放电优先于电网购电）。每条流向都有明确的经济意义，支撑了后续多维度收益分析。

### 3.3 收益计算口径

**经营总收益公式**:
```
经营总收益 = 光伏上网收入 + 充电桩收入 + 储能供工厂省电 + 储能供充电桩收入 - 电网购电成本
```

**储能额外收益公式**:
```
储能额外收益 = 含储能经营总收益 - 无储能经营总收益
```

其中 `无储能` 场景通过 `buy_no = 真实总负载 - 光伏发电` 计算（无储能时所有缺口都由电网补足）。

**评价**: "有/无储能"对比法是评估储能增量价值的行业标准方法。公式中 `工厂省电收益` 只统计光伏直供工厂部分，储能供工厂的收益通过电网购电成本下降体现，避免了重复计算。

### 3.4 多场景并行计算

项目对三种光伏上网电价（0.1、0.2、0.35 元/度）并行计算收益，反映的是同一套物理数据在不同电价政策假设下的收益差异。这为业主提供了政策敏感性分析能力。

### 3.5 分时电价自动识别

**代码位置**: [pricing.py L84-120](file:///Users/cai/SynologyDrive/Project/#ProjectWork-000000-光伏收益计算/Script/pricing.py#L84-L120)

通过电价数值排序自动映射谷/平/峰/尖标签，而非硬编码时间段：

| 价格档位数 | 映射规则 |
|---|---|
| 1 档 | 全部为"平" |
| 2 档 | 最低价=谷，最高价=峰 |
| 3 档 | 最低价=谷，中间=平，最高价=峰 |
| 4 档 | 最低价=谷，次低=平，次高=峰，最高=尖 |
| 5+ 档 | 最低=谷，次低=平，最高=尖，其余全部为峰 |

**评价**: 设计巧妙，电价政策变化时只需更新 CSV 数据，无需修改代码。这在实际运维中大幅降低了维护成本。

## 四、光伏数据提取评价

### 4.1 OCR 自动标定

**代码位置**: [extract_and_merge_pv.py L52-139](file:///Users/cai/SynologyDrive/Project/#ProjectWork-000000-光伏收益计算/Script/extract_and_merge_pv.py#L52-L139)

工作流程：
1. 使用 Tesseract OCR 识别截图纵轴刻度数字
2. 过滤：排除非左侧区域、低置信度、非数字结果
3. 对识别到的刻度值与像素 Y 坐标做线性拟合（`np.polyfit`）
4. 推导出 `y_zero`（功率=0 的像素 Y 坐标）和 `px_per_100kw`（每 100kW 对应的像素数）

**评价**: 解决了不同截图缩放/分辨率导致像素比例不一致的问题。选择线性拟合是合理的，因为坐标轴本身就是等距刻度。

### 4.2 曲线提取与插值

1. 在 HSV 空间中提取绿色曲线像素
2. 对每个 X 坐标取 Y 坐标平均值（曲线有宽度）
3. 使用 `scipy.interpolate.interp1d` 线性插值到 5 分钟时间步长
4. 可选地按人工校准值整体缩放

**评价**: 方法实用。线性插值在 5 分钟粒度下精度足够。校准机制提供了人工兜底能力，双保险设计降低了单一方案失败的风险。

### 4.3 数据校准机制

`数据/数据校准.csv` 记录了人工确认的每日光伏总发电量，用于在 OCR 提取结果偏差较大时进行校准。该校准是可选的（需 `--use-calibration` 参数），默认不启用。

**评价**: 务实设计。OCR 在大多数情况下可靠，但特殊截图（如裁切、低分辨率）需要人工校准兜底。

## 五、年化预测与 ROI 分析评价

### 5.1 预测方法

**代码位置**: [annual_prediction.py](file:///Users/cai/SynologyDrive/Project/#ProjectWork-000000-光伏收益计算/Script/annual_prediction.py)

1. 按晴天/多云/阴雨分组，从样本天 CSV 计算各类天气的日均收益
2. 加权求和：`年收益 = 晴天天数 × 晴天日均 + 多云天数 × 多云日均 + 阴雨天数 × 阴雨日均`
3. 增量法计算储能价值：`储能年收益 = 有储能年收益 - 无储能年收益`

**天数假设**: 晴天 130 天、多云 97 天、阴雨 138 天（合计 365 天，符合汕头亚热带气候特征）

### 5.2 ROI 计算

```
ROI = (年收益 / 投资额) × 100%
静态回收期 = 投资额 / 年收益
```

| 资产 | 投资额 | ROI 范围 | 回收期范围 |
|---|---|---|---|
| 光伏+充电站 | 175 万元 | 26.6%~29.3% | 3.4~3.8 年 |
| 第 1 个储能柜 | 22 万元 | 29.3%~35.7% | 2.8~3.4 年 |
| 第 2 个储能柜 | 22 万元 | 16.5%~17.5% | 5.7~6.1 年 |

### 5.3 ROI 报告的商业洞察质量

ROI 报告中的 Q&A 分析部分质量较高：

- **边际效用递减分析**: 准确指出第 2 台储能 ROI 骤降的原因是谷段充电时长和峰段消纳空间已被第 1 台充分利用。
- **储能容量瓶颈分析**: 正确识别了"谷段充满 → 中午光伏余电无法消纳"的调度冲突。
- **"两充两放"策略建议**: 提出了通过差异化定价释放储能容量、实现日内双循环的运营策略，具有实际操作价值。

**评价**: 报告不仅给出了数字结果，还提供了深度的商业洞察和可操作的策略建议，这是纯技术分析所欠缺的。

### 5.4 储能增量拆解报告

**代码位置**: [extra_ess_analysis_report.py](file:///Users/cai/SynologyDrive/Project/#ProjectWork-000000-光伏收益计算/Script/extra_ess_analysis_report.py)

该脚本对 1 台 vs 2 台储能进行逐 5 分钟的仿真对比，拆解出：
- 电网购电成本变化（负载侧 + 充电侧）
- 光伏余电吸纳变化
- 充放电循环损耗
- 分时段增量明细

**评价**: 这是对 `annual_prediction.py` 的有效补充。年化报告给出"要不要加"的结论，拆解报告回答"为什么加了之后收益不如预期"的问题，二者配合完整支撑投资决策。

## 六、数据处理质量评价

### 6.1 离散积分精度

5 分钟瞬时功率 × 5/60 小时的矩形积分方法在新能源计算中是行业标准做法。对于光伏发电和储能充放电这类变化相对平缓的曲线，5 分钟粒度的矩形积分误差在可接受范围内。

### 6.2 电价版本化管理

`电网电价.csv` 和 `充电桩定价.csv` 均支持多条记录，按"记录日期 ≤ 目标日期"自动选择最新版本。这使得电价政策调整后，历史日报重生成时仍能使用当时的正确电价。

### 6.3 数据校验

- 光伏功率强制 `max(0, ...)`，排除负值
- 真实总负载强制 `max(0, ...)`，排除负值
- 充电桩用电 `clip(lower=0)`
- OCR 结果过滤低置信度和非数字

**评价**: 基本的数据边界检查已覆盖，但缺少对 CSV 列缺失、数据异常跳变等情况的显式处理。

## 十、改进建议汇总

### 高优先级

1. **单元测试**: 为 `process_data`、`calc_profit_for_price`、`simulate` 等核心函数编写回归测试，使用已知输入输出校验计算结果
2. **异常处理**: 增加 CSV 列缺失、空数据、异常跳变等情况的检测和有意义的错误提示
3. **工厂负载模型**: 当前固定 50kW 硬编码时间段，建议支持从文件加载实际负载曲线

### 中优先级

4. **日志系统**: 用 `logging` 替代 `print`，支持日志级别和文件输出
5. **年化预测精细化**: 引入月度电价变化、日照时长季节变化等因素
6. **储能策略仿真**: 扩展为更智能的调度策略（如 SOC 预留、两充两放策略仿真）
7. **代码清理**: 提取散落的魔法数字到配置区域，消除 `sys.path.append` 模式

### 低优先级

8. **报告格式升级**: 考虑生成 HTML 或 PDF 格式报告，便于非技术人员阅读
9. **可视化**: 增加图表输出（如能量流桑基图、逐时功率曲线对比）
10. **数据备份**: 自动备份修改前的 CSV 文件，避免数据丢失
