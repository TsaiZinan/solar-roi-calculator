# 每日分析 (Daily Analysis)

当用户请求执行“每日分析”时，请作为全自动的数据分析和经济效益测算专家，严格按照以下三个阶段执行任务：

## 第一阶段：检查与预处理数据

1. **检查文件是否存在**：
   - 目标目录：`数据/`
   - 目标文件：当日的 CSV 日报表（以 `日报表_` 开头）和光伏曲线图片（**图片以日期命名**，如 `20260419.png` 或 `20260419.jpg`）。
   - **如果缺少其中任何一类文件，请立即停止执行并提醒用户上传缺失的文件。**

2. **文件归类**：
   - **读取 CSV 文件内部的数据**，以表格中“时间”列实际记录的日期（例如 `20260421`）为准，而非依赖文件名。
   - 在 `数据` 目录下创建以该真实数据日期命名的文件夹（如 `20260421/`）。
   - 将上述 CSV 文件和对应的 PNG 图片移动到该日期文件夹中。

3. **数据提取与合并**：
   - 首先检查 CSV 文件中是否已存在 `光伏发电功率(kW)` 列且有有效数据。如果已有该数据，则**直接跳过图片处理**，进入下一阶段。
   - 如果没有该数据，则执行 Python 脚本提取并合并光伏数据：
     ```bash
     python Script/extract_and_merge_pv.py <PNG图片绝对路径> <CSV绝对路径>
     ```
   - 确认脚本执行成功，且 CSV 文件中已成功新增一列 `光伏发电功率(kW)`。
   - **注意**：必须在确保光伏数据和原有储能、负载数据对齐合并后，才能进入下一阶段的分析。

## 第二阶段：系统运行规则梳理

分析合并后的 CSV 数据（每5分钟一行，每行为一个时间步长 = 5/60 小时），基于以下系统运行规则和电价体系建立测算模型：

### 1. 运行策略
- **主动充电**：00:00 - 08:00（谷段），从电网购买低价谷电将储能充满。
- **被动充电（光伏消纳）**：除了 00:00 - 08:00 以外的时间，储能仅使用光伏的**多余电量**充电，不从电网买电。
- **放电策略**：优先满足工厂用电和充电桩用电，以最大化减少高峰期从电网购电。
- **特殊规则（阴天补电）**：如果当天为阴天，中午时段会执行从电网购买电量补电的操作（具体时段和补电量需在分析时视实际数据而定）。

### 2. 电价体系（元/度）
- **电网购电成本**：需从 `数据/电网电价.csv` 中读取对应时间的定价进行计算。该表第一列为记录日期，每次有定价变化则新增一行，后续列为00:00至23:00每小时的定价。匹配逻辑为：寻找“记录日期”小于等于当前测算日期的最新一条记录（若找不到则取最早的一条），然后根据当前时间段匹配对应小时的单价。
- **充电桩对外卖电**：需从 `数据/充电桩定价.csv` 中读取对应时间的定价进行计算。该表第一列为记录日期，每次有定价变化则新增一行，后续列为00:00至23:00每小时的定价。匹配逻辑与电网购电成本相同。
- **光伏余电上网（卖给电网）**：统一收购价 `0.10`。

### 3. 负载拆分逻辑
CSV 中的 `负载功率(kW)` 为系统总负载。需要将其拆分为：
- **工厂日常用电**：07:00 - 12:00 及 13:00 - 18:00（下午1点到6点），固定为 50kW（即每小时50度）。其余时间为 0。
- **充电桩用电**：实际的 `负载功率(kW)` 减去工厂用电（若相减为负数，则按 0 算）。

## 第三阶段：数据分析与收益测算

1. **逻辑推演**：根据上述规则，计算系统在每一时刻的能量流向优先级（光伏发出的电优先给负载 -> 多余给储能充电 -> 再多余卖给电网；电不够时优先放储能 -> 储能用完买市电），进而计算每一时刻的资金流水（买电花了多少、充电桩赚了多少、卖光伏电赚了多少）。
2. **多场景测算**：需同时计算光伏上网电价为 `0.1`、`0.2` 和 `0.35` 三种情况下的收益差异，并计算**增加该储能系统后能额外带来的收益**（即“有储能系统的总收益”减去“无储能系统情况下的光伏直发直用及充电桩收益”）。

3. **生成收益报表文件**：
   - 按照分时（峰、平、谷）生成包含光伏产电、充放电、向电网买卖电等核心指标的基础报表。
   - **生成光伏发电的流向与时间分布（分时段统计：直接消纳、充入储能、余电上网）**。
   - **生成电网购电的流向与时间分布（分时段统计：满足负载、充入储能）**。
   - **生成按小时汇总的详细基础报表（00:00 - 23:00，每小时一行）。**
   - 给出明确的“最终收益结论”及“储能系统额外收益”。
   - 将上述所有结果导出至 Markdown 文件，放置在 `报告/` 文件夹里，文件名必须包含日期（如 `每日收益分析报告_20260421.md`）。
   - **自动更新总表**：每次生成每日报告后，务必确保将当天的核心指标（包含三种光伏电价 0.1、0.2、0.35元/度 对应的“总收益”和“储能额外收益”）自动追加到 `报告/总收益分析报表.md` 的末尾，并告知用户总表已更新。
   - **自动推送到 GitHub**：每日分析和报表更新完成后，必须自动执行 `git add .`、`git commit -m "Add daily analysis for <日期>"` 和 `git push`，将新增的分析结果和数据文件上传到 GitHub。
