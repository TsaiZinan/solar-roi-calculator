# 光伏收益计算

本项目用于计算光伏、储能与充电桩一体化系统的每日收益、历史汇总收益和全年收益预测。

项目当前重点解决三类问题：
- 基于日度 CSV 数据生成每日报告
- 自动汇总多天日报形成总收益分析报表
- 在统一参数配置下进行全年收益与 ROI 预测

## 快速开始

如果你只是日常使用，推荐直接通过 AI 技能执行：
- 在对话中输入 `每日分析`
- AI 会自动检查数据目录、合并光伏数据、生成每日报告并更新总表

如果你是维护者，最常用的脚本入口如下：
- `Script/calc_revenue.py`：生成单日收益分析报告
- `Script/regenerate_reports.py`：批量重生成历史日报
- `Script/init_summary.py`：按现有日报重建总表
- `Script/annual_prediction.py`：生成年化收益和 ROI 报告

## 目录结构

- `Script/`：核心脚本与统一配置
- `数据/`：输入数据目录，包含日度文件夹和电价表
- `报告/`：输出报表目录
- `.trae/skills/daily-analysis/`：AI 技能说明

## 核心脚本

- `Script/config.py`
  - 统一存放路径、工厂负载、光伏电价场景、储能参数、年化预测参数、ROI 报告名等配置
- `Script/pricing.py`
  - 从 `数据/电网电价.csv` 和 `数据/充电桩定价.csv` 读取价格
  - 按价格档位自动识别 `尖/峰/平/谷` 标签
- `Script/calc_revenue.py`
  - 主收益计算脚本
  - 生成单日 Markdown 报告
  - 生成单日报后自动重建总表
- `Script/init_summary.py`
  - 从历史日报中提取经营总收益和储能额外收益
  - 重建 `报告/总收益分析报表.md`
- `Script/regenerate_reports.py`
  - 批量扫描 `数据/20*/` 下的日度 CSV
  - 逐日重生成日报和总表
- `Script/annual_prediction.py`
  - 基于样本天和配置参数推算全年收益
  - 输出 ROI 报告
- `Script/extract_and_merge_pv.py`
  - 从光伏曲线图片中提取发电曲线并写回 CSV

```text

```

### 必需文件

- 日度 CSV：来自阳光云导出的 5 分钟间隔报表
- 光伏截图：用于光伏曲线提取与后续复盘归档

### 电价文件

- `数据/电网电价.csv`
  - 记录每次电网购电价格调整后的 24 小时电价
- `数据/充电桩定价.csv`
  - 记录每次充电桩对外售电价格调整后的 24 小时电价

脚本会自动选择“记录日期小于等于目标日期的最新一条记录”作为当天电价。

## 核心计算口径

### 真实总负载

项目当前口径不是直接把 CSV 中的 `负载功率(kW)` 当作真实总负载。

实际使用公式为：

```text
真实总负载 = 光伏发电功率 - 负载功率
```

在此基础上再拆分为：
- 工厂用电
- 充电桩用电

### 分时标签

分时标签不是按固定时间段写死，而是根据 `电网购电价` 的价格档位自动识别：
- 最低价对应 `谷`
- 次低价对应 `平`
- 更高价对应 `峰`
- 最高价在存在第四档时对应 `尖`

因此不同月份、不同电价政策下，时段标签和对应小时都可能变化。

### 收益口径

当前报告口径为：

```text
经营总收益 = 现金类收益 + 工厂省电收益
储能额外收益 = 含储能经营总收益 - 无储能经营总收益
```

其中：
- 现金类收益：充电桩收入 + 光伏上网收入 - 电网购电成本
- 工厂省电收益：光伏直发直用替代工厂购电所节省的成本

### 2026-04 Bug 修复记录

- 修复时间：`2026-04-25`
- 问题现象：旧版脚本在计算 `工厂省电收益` 时，直接按工厂固定负载 `50kW` 全额乘以对应购电电价计入，等于默认工厂负载在工厂运行时段全部被光伏替代。
- 实际影响：在光伏不足、工厂大量依赖储能或电网供电的日期，会明显高估 `工厂省电收益` 和 `经营总收益`。
- 修复口径：现已改为仅统计“实际由光伏直供工厂负载”的那部分电量，再乘以对应时段电网购电价格。
- 报告表述同步调整：`光伏纯卖电收益` 已改名为 `光伏全额上网理论值`，并明确标注“仅作对比，非当日实际可实现收益”。
- 数据修复动作：已使用 `python3 Script/regenerate_reports.py` 对全部历史日报和 `报告/总收益分析报表.md` 执行全量重生成。

### 储能效率

- `calc_revenue.py` 使用的是实测储能功率数据，不额外再套一层 `0.95` 效率
- `annual_prediction.py` 属于仿真推演，会使用配置中的储能效率参数

## 常见问题

### 为什么报告中的充放电量和储能软件显示值会有微小差异？

主要原因包括：
- 5 分钟瞬时功率积分本身存在离散采样误差
- CSV 多为交流侧功率，软件可能统计直流侧吞吐量
- 辅助设备损耗在不同系统中的计入口径不同

这类误差通常属于合理范围。若要进一步逼近设备原始统计值，需要数据源提供连续累计电量字段。

### 为什么有时日报和总表会不一致？

常见原因是：
- 脚本逻辑已更新，但历史日报还没按新口径重生成
- 总表只是对现有日报进行汇总，因此日报不更新，总表也不会自动变成新口径

解决方式：
- 先执行 `python3 Script/regenerate_reports.py`
- 再检查 `报告/总收益分析报表.md`

### 为什么有的月份会出现“尖”时段，有的没有？

因为脚本是按电价档位自动识别时段，而不是按固定时间硬编码。

只要当日电价表中存在第四档更高价格，报告里就会出现 `尖`。
